## Cross-Phase Integration Issues

### HIGH: publish-skills Token Resolution (SKILL-10, REG-10)

**File:** `skills/publish-skills/SKILL.md`, Step 4 code template
**Issue:** `TOKEN = os.environ["REGISTRY_TOKEN"]` raises `KeyError` when env var absent
**Impact:** Users who configure per-registry tokens in `registries.json` (the designed multi-registry workflow) cannot publish. Only users who set the `REGISTRY_TOKEN` env var globally can publish.
**Fix:** Change to `TOKEN = os.environ.get("REGISTRY_TOKEN") or REGISTRY_ENTRY_TOKEN` where `REGISTRY_ENTRY_TOKEN` comes from Step 1's registries.json resolution.

### MEDIUM: Project ID Cache Path Never Written (OBS-05)

**File:** `lib/commands.py` `cmd_init()`
**Issue:** `capture.py` checks `.claude/.prism_project_id` for fast project ID lookup, but `cmd_init()` never writes this file.
**Impact:** Every hook invocation runs `git remote get-url origin` subprocess (~5ms). Functionally correct but bypasses the designed cache acceleration.
**Fix:** Add `(Path.cwd() / ".claude" / ".prism_project_id").write_text(project_id)` in `cmd_init()`.

### MEDIUM: PRISM_PROJECT vs PRISM_PROJECT_ID (OBS-05)

**File:** `lib/commands.py` `_setup_hooks_and_mcp()`
**Issue:** MCP server env set as `PRISM_PROJECT`, but `capture.py` and `project.py` check `PRISM_PROJECT_ID`. Different variable names — hook shortcut is dead code.
**Impact:** Same as above — hooks always fall through to git detection.
**Fix:** Align env var names or rely on cache file fix.

## Broken E2E Flows

### publish-skills with Per-Registry Token

**Flow:** engram -> promote -> skill -> publish-skills -> registry Worker
**Break point:** Step 4 token resolution
**Steps:**
1. User runs `/publish-skills` -> resolves registry from registries.json (PASS)
2. Computes delta via .published.json content hashes (PASS)
3. Builds payload with changed skills (PASS)
4. **`TOKEN = os.environ["REGISTRY_TOKEN"]` -> KeyError** if env var not set (BROKEN)
5. POST to registry Worker API (NEVER REACHED)

**Workaround:** Set `REGISTRY_TOKEN` env var before running `/publish-skills`.
