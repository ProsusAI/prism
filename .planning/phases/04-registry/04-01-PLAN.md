**Constants:**

**Core CRUD functions:**

1. `load_registries() -> dict` — Load registries.json. If file does not exist, check config.json for `registry_url` and auto-migrate per D-02: create a registry entry named `"default"` with that URL, `writable=True`, empty token. Use `save_registries()` for atomic write. If neither exists, return `{"registries": [], "default": None}`. Handle JSONDecodeError gracefully.

2. `save_registries(data: dict) -> None` — Atomic write: write to `str(REGISTRIES_PATH) + ".tmp"`, then `os.rename(tmp, str(REGISTRIES_PATH))`. After rename, set `os.chmod(str(REGISTRIES_PATH), 0o600)` per security requirement (tokens in plaintext). Create parent dir with `mkdir(parents=True, exist_ok=True)`.

3. `add_registry(name: str, url: str, token: str = "", writable: bool = True) -> None` — Load registries, check name doesn't already exist (raise ValueError if duplicate), append entry dict `{"name": name, "url": url.rstrip("/"), "token": token, "writable": writable}`, if no default set yet make this the default, save.

4. `remove_registry(name: str) -> None` — Load registries, find and remove entry by name (raise ValueError if not found). If removed entry was the default, set default to first remaining registry name or None if empty. Save.

5. `list_registries() -> list` — Load registries, return list of dicts. Each dict has keys: name, url, token (masked: first 8 chars + "..." if len > 8 else "***"), writable, is_default (bool).

6. `set_default_registry(name: str) -> None` — Load registries, verify name exists in entries (raise ValueError if not), set `data["default"] = name`, save.

7. `get_registry(name: str) -> dict` — Load registries, find entry by name, return it. Raise ValueError if not found.

8. `get_default_registry() -> dict | None` — Load registries, return entry matching data["default"], or None if no default.

9. `generate_token() -> str` — Return `"prism_" + secrets.token_hex(32)`. Import secrets at top of file.

**Token resolution (backward compat):**

10. `resolve_token(registry: dict) -> str` — Check `os.environ.get("REGISTRY_TOKEN")` first (backward compat with Phase 3), then fall back to `registry.get("token", "")`. This ensures existing REGISTRY_TOKEN env var users are not broken.

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| CLI input -> registries.json | User-supplied registry names, URLs, tokens written to local config |
| registries.json -> file system | Tokens stored in plaintext on disk |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-01 | Information Disclosure | registries.json token storage | mitigate | `os.chmod(str(REGISTRIES_PATH), 0o600)` after every write in `save_registries()`. Mask tokens in `list_registries()` output (first 8 chars + "..."). |
| T-04-02 | Tampering | Registry name injection | mitigate | Validate registry name is kebab-case `[a-z0-9][a-z0-9-]*[a-z0-9]` in `add_registry()` before writing to registries.json. |
| T-04-03 | Spoofing | Token generation weakness | mitigate | Use `secrets.token_hex(32)` (cryptographic RNG), never `random`. Prefix with `prism_` for identifiability. |
| T-04-04 | Information Disclosure | Token in CLI output | accept | `prism registry token create` must display the token once for the user to copy. This is unavoidable -- same pattern as `gh auth token`. Terminal scrollback is user's responsibility. |
</threat_model>
