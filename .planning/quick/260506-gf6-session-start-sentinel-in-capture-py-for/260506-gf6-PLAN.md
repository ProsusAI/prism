```python
def _spawn_background(prism_home: Path, args: list) -> None:
    """Spawn a background prism command. Non-blocking, fire-and-forget."""
    import shutil
    import subprocess

    prism_cli = shutil.which("prism")
    if not prism_cli:
        candidate = prism_home / "prism"
        if candidate.exists():
            prism_cli = str(candidate)
    if not prism_cli:
        return

    try:
        with open(os.devnull, "w") as devnull:
            subprocess.Popen(
                [sys.executable, prism_cli] + args,
                stdout=devnull,
                stderr=devnull,
                start_new_session=True,
            )
    except OSError:
        pass
```

1. **Sentinel path**: `/tmp/prism_synced_{project_id}_{date}` where `date` is

   return immediately. NO `os.stat()`, NO `open()`, NO `read()`. This is the hot path
   on every PreToolUse after the first.

3. **Slow path (sentinel absent, first call of the day)**:
   - Create the sentinel atomically: open with `O_CREAT | O_EXCL | O_WRONLY`, close
     immediately. If `FileExistsError` (race with another concurrent capture), return
     silently — another process already won the race.
   - On any other `OSError` (e.g., `/tmp` not writable), return silently — DO NOT crash.
   - After successful sentinel creation, spawn `prism sync --project {project_id}`
     using the existing `_spawn_background(prism_home, ["sync", "--project", project_id])`
     helper. Reuse it — do NOT duplicate spawn logic.

4. **Wire into `main()`**: Add a single call to `_check_session_sync(prism_home, project_id)`
   immediately BEFORE the existing `_check_triggers(...)` call (around line 71). The
   sync gate should not depend on observation count.

5. **Cleanup of stale sentinels**: Before the `os.path.exists` check, opportunistically
   clean stale sentinels from previous days. Implement as: if sentinel name pattern for
   THIS project exists with yesterday's date or older, `os.unlink()` it. Keep this cheap:
   only attempt cleanup once per main() call, wrapped in try/except OSError. Use a
   simple glob via `os.listdir("/tmp")` with prefix filter `f"prism_synced_{project_id}_"`
   and string comparison against today's date suffix. If `/tmp` listing fails, skip
   cleanup silently.

   IMPORTANT: cleanup must NOT delay the fast path. Order is:
   a. Compute today's sentinel path
   b. Check `os.path.exists(today's path)` — if True, return immediately
   c. Only if sentinel absent, run cleanup of stale ones for this project, then
      attempt creation + spawn

6. **Wrap entire helper in try/except Exception** at the outermost level — capture.py
   must never crash. Mirror the pattern in `main()` line 230-232 (the bare except).

7. **No new top-level imports**. `subprocess`, `os`, `sys`, `Path`, `datetime`,
   `timezone` are already imported. Keep `_spawn_background` reuse so `shutil` and
   `subprocess` stay lazily imported there.
